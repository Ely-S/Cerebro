# Hardware Component Verification
Project: Cerebral Oximeter (Prototype 1)  
Date: 2026-02-28  
Basis: `spec.txt` Rev 3.2, `BOM.txt`, `validation.txt`

## 1) Purpose
This document records hardware component-level verification using datasheets and authoritative online sources.  
It complements (does not replace) bench verification requirements in `validation.txt`.

## 2) Status Labels
- VERIFIED: Component choice is supported by source specs and is compatible with design intent.
- PARTIAL: Usable source exists, but source is secondary/non-manufacturer or incomplete for exact MPN.
- BENCH-REQUIRED: Datasheet compatibility is acceptable, but requirement closure depends on bench testing.
- N/A: Mechanical/tooling item with no formal electrical datasheet requirement.

## 2A) Source Documentation
- Optics source provenance is centralized in `datasheets/optics_source_register.txt`.
- Numeric optics decisions prioritize manufacturer datasheets over product-page summaries or mirrors.
- Any optics claim without direct primary citation is marked unverified/pending.

## 4) Requirement-Level Closure Impact
Component datasheet compatibility is sufficient for design-level confidence, but does not close bench-dependent requirements.  
Per `validation.txt`, final closure still requires bench execution for:
- ADC address/config readback
- TDM timing capture
- optical safety measurement under final TDM timing (see ISSUE-002 below)
- precision/statistical claim (+/-2% at 95% confidence)
- optical isolation and dark-test performance

## 5) Cross-Verification Issue Log (2026-03-01)

The following issues were identified during an optics cross-verification pass against the
SFH4253B datasheet (PDF v1.2, 2025-12-01). All issues are also recorded in
`datasheets/optics_validation.txt` and `validation.txt`.
Source provenance for optics claims is documented in `datasheets/optics_source_register.txt`.

ISSUE-001 [MINOR -- CORRECTED]
  File    : datasheets/optics_validation.txt
  Finding : 850nm Vf pre-check band was stated as 1.0-1.8V. V&V plan Test 1.1 (authoritative)
            specifies 1.0-1.9V. Datasheet max VF at 70mA is 1.90V, so 1.9V is correct.
  Fix     : Upper bound updated to 1.9V in optics_validation.txt.
  Impact  : None. Typical VF values (1.55V) are well within both bounds.

ISSUE-002 [RESOLVED]
  Files   : validation.txt (R-SAFE-01, NCR-001), datasheets/optics_validation.txt
  Finding : Legacy optical safety calculation was tied to the old 36 Ohm design case
            (~89mA), which exceeded the project criterion on paper.
  Fix     : R-SAFE-01 verdict changed from OPEN to PASS (conditional) in validation.txt.
            Changed 850nm series resistor from 36 Ohms to 68 Ohms. Current drops to ~47mA,
            peak power to ~36-40mW, and average power to ~9-10mW in 4-state TDM.
  Impact  : Resolved. Human-subject demonstration still requires bench verification.

ISSUE-003 [MINOR -- CORRECTED]
  File    : validation.txt, Section 2 (R-DRV-01 calculation)
  Finding : 850nm current calculation used VF = 1.5V (unspecified). Datasheet gives VF typ
            = 1.60V at 100mA (explicit table entry).
  Fix     : Calculation updated to final drive configuration:
            I = (5.0 - 0.2 - 1.60) / 68 = 47.1mA.
  Impact  : R-DRV-01 verdict unchanged (47.1mA is within 40-50mA band). No design change.
